## 1. 측정 방법론 (이 캠페인의 최대 수확)

### 1.1 전이법칙 프레임워크 — "오프라인 +는 LB +가 아니다"
모든 오프라인 게인을 **기전 클래스**로 분류하고 클래스별 LB 전이계수(β)를 실측했다:

| 클래스 | 기전 | 전이 실측 |
|---|---|---|
| Class-R | 강건화·지표정합 (FGM, soft-F1) | **전이함** (+0.0037, +0.00085) |
| Class-A | 입력분포 복원 (gen_rescue) | 전이하나 **소진됨** (β≈0.82, 1회성) |
| Class-M | 적합 강화 (에폭↑, 증류, warm-start) | **역전** (−0.005~−0.009) |
| Class-C | 결정규칙 (가중치·게이트·결합기·bias refit) | **0/12 전멸** — fold0 +0.005급도 LB 음전 |

**적용법:** 새 아이디어가 나오면 먼저 "어느 클래스인가"를 묻고, 사망 클래스면 오프라인 수치가 아무리 좋아도 발사하지 않는다.

### 1.2 승자저주 정량화 — 최고점은 μ가 아니다
- 은행 멤버(0.7911)는 동일 레시피 재추첨 5발 실측 결과 **+2.7σ 행운 드로우**였다 (드로우 진실 μ≈0.7862, σ≈0.0018).
- "최고 레시피를 재현하면 최고 점수가 재현된다"는 착각이 밤새 GPU 슬롯 5개를 소모했다.
- **적용법:** 어떤 구성의 기대치를 논하기 전에 (a) 동일시드 재현 노이즈 ε, (b) 시드간 σ, (c) 관측점이 몇 번째 드로우의 max인지를 먼저 추정하라.

### 1.3 in-dist 평가 ↔ 히든 일반화 역상관 실증
- 5k in-dist 평가에서 최고였던 멤버(d64_6, 0.82499)가 LB 최저(0.7833)였다.
- 학습에 쓴 데이터로는 선별이 불가능하며, 오히려 역선택 위험이 있다.
- **적용법:** in-dist 수치는 "파이프라인이 깨졌는지" 판별(버그 vs 품질)에만 쓰고, 시드/변형 선별에는 절대 쓰지 않는다.

### 1.4 사전등록(pre-registration) 규율
- 판독 **전에** 분기표를 등록한다: "d4≥X면 A, ≤Y면 B" + 잭팟 임계 + 게이트 3조건(raw: Δ5k≥+0.0015 ∧ LCB>0 ∧ LB조건).
- 효과: 충격 판독(0.7861 동률) 직후에도 감정적 이탈 없이 기계적으로 분기 실행.
- 등록된 것에서 벗어날 때는 "사전등록 일탈"로 명시하고 근거를 1줄 기록.

### 1.5 통계 위생
- 단일 LB 점으로 결론 금지: 부호검정(2/2 양전 ≠ 증명, P=25%), paired bootstrap LCB, 소표본 t.
- "3연속 상승" 같은 패턴은 순서통계 확률(1/3!=17%)부터 계산.
- 동률(Δ0.00006) 같은 이례 신호는 우연확률을 명시적으로 계산한 후 수사 개시.

## 2. 발견한 기술 레버 (도메인 지식)

1. **soft-F1 서로게이트 손실**: macro-F1 지표정합 손실(배치 per-class soft TP/FP/FN → 1−F1)을 CE에 w=0.5로 가산. **칼날 최적** — w0.7/클래스가중/AWP결합 전부 −0.003 이상 붕괴. 지표정합은 유일하게 전이한 "학습" 레버.
2. **SR 재양자화 지터포켓 (신규 발견)**: 배포 멤버의 fp16 원본을 확률적 반올림(비편향)으로 재양자화하면 int8의 ~25%가 ±1LSB 플립 → 저마진 예측 ~3% 플립 = **GPU 없이 무한 생성 가능한 재추첨 티켓**. RTN이 저추첨이었으면 공짜 회수(+0.0005 실현). σ≈0.0002로 작지만 μ≥원본. **m1+m3 복합 > m1 단독** (3/3 확증, +0.0003). 증폭(amp≥2)은 μ비용 발생 — k=1이 최적.
3. **FGM 적대학습**이 최대 단일 레버(+0.0037). 강건화 계열이 OOD 히든에서 살아남는다.
4. **입력복원(gen_rescue)**: 절단으로 소실된 구조 헤더를 보존하는 직렬화 수리 — 입력분포 정합이 아키텍처 다양화보다 컸다(+0.0031).
5. **캐스케이드 서빙**: 주멤버 마진≥0.85면 솔로, 아니면 3아키 앙상블 + macro-F1 bias 후처리(+0.0103). 단 이 결정규칙층의 재튜닝은 전이 0/12.
6. **gradckpt 소동의 교훈**: "GC런만 낮다"는 완벽한 상관도 클린 대조실험 2발로 무죄 판명 — 상관≠인과, 판별실험 설계가 우선.

## 3. 운영 하네스 (그대로 복붙 가능한 인프라)

### 3.1 제출물 조립 파이프라인 (`sim/assemble_*.sh`)
필수 가드 6종 — 전부 실제 사고 후 도입됨:
1. **자기파괴 가드**: `[ "$DST" != "$SRC" ]` (은행을 rm -rf할 뻔한 사고)
2. **캐시 신선도**: q8 캐시가 원본 zip보다 오래되면 재생성
3. **메타 assert**: run_meta의 서빙 플래그(mht 등)를 조립마다 검증
4. **byte-diff 가드**: 새 티켓이 기존과 바이트동일이면 FATAL (슬롯 낭비 방지)
5. **용량 가드**: `bytes < 1e9` (MiB≠MB 사고: du의 879M는 사이트의 920.9MB)
6. **5행 런타임 캐너리 필수**: 실제 제출 zip을 풀어 script.py를 끝까지 실행, missing=0 + 클래스 유효성 검증 — import 누락(xrestore ModuleNotFoundError)으로 슬롯 2개 태운 후 의무화

### 3.2 장시간 학습 감시
- **PID 기반** 감시 (커맨드라인 매칭은 env변수를 못 봐서 오경보 냈음)
- 감시 항목: member zip 출현(=완주) / PID 소멸(Z 포함) / 로그 mtime 정지(stall) / Traceback / CUDA 생존 폴링
- **zip 완결성 대기**: 출현 즉시 열지 말고 크기 안정 + `zipfile.testzip()` 통과까지 대기 (BadZipFile 레이스 실사고)
- NVML 붕괴 시: 기존 CUDA 컨텍스트는 생존할 수 있다 — **재시작 전에 완주 대기**가 정답이었음

### 3.3 GPU 함대 운영 수칙
- kill은 반드시 PID 지정 (패턴 kill 금지)
- 이중발사 감지: 같은 TAG 프로세스 2개 = GPU 반분 낭비 — 환경변수 덤프(`/proc/PID/environ`)로 정체 확인 후 후발 제거
- 자동 큐의 busy() 결함이 학습 중인 GPU에 오발사 — **마감 임박 시 자동화를 죽이고 수동 원샷 스크립트로** (`launch_post_restart.sh`, 에폭 자동룰 + 90초 생존확인 내장)
- 컨테이너 재시작 후: CLI 심링크(코덱스 등)가 확장 업데이트로 끊길 수 있음

### 3.4 멀티에이전트 판정 하네스
- **매 판독마다** 다관점 패널(법의학 / 재계획 / 반증사냥) + 이견은 실측근거 우선으로 재정
- **교차검증 이원화**: Claude 패널 ↔ codex 독립판정을 상호 비판시켜 수렴 (서로 다른 오류를 냈고, 교차에서 잡혔음)
- 적대적 판정 스키마: 각 아이디어에 SUBMIT/HOLD/DEAD + expected_delta + 근거 — "죽은 계열 재개봉 금지"를 기계적으로 집행
- 위기 프로토콜: 충격 판독 → ①즉석 무결성 검사(해시/메타) → ②저비용 판별 프로브(500행 라벨) → ③분리 실험(단일변수) → ④가설별 사전확률 + 판독별 업데이트 규칙 등록 → ⑤분기 실행

### 3.5 실험 원장 (`experiments_master.csv`)
- 모든 판독·판정·사고를 단일 CSV에 append (LB실측/DIAG/VERDICT/PLAN 태그)
- 에이전트들이 이 원장을 읽고 사망계열을 자동 대조 — "이미 죽은 아이디어의 재제안"을 구조적으로 차단
- **아쉬웠던 것: 학습 로그에 loss 계기가 없어** 0.786 사고의 법의학이 실명 상태였다. 다음엔 에폭당 loss 1줄부터.

### 3.6 슬롯(제출권) 경제학
- 티켓별 P(컷)·P(경신)·기대폭을 정량화해 **버림서열**을 사전 확정
- 예비 슬롯 규율: 마감 T-10분까지 오류보험 보존 → 무사고 시 필러 전환 (전환 순서도 사전등록)
- **체크박스 불변식**: 최종선택은 마감 직전 1회가 아니라 "항상 현재 전역최대를 가리키게" 상시 유지, 갱신은 풀 자릿수 엄격초과 + 제출ID + 파일명 3중 대조, 새로고침 재확인 + 스크린샷
- 점수대가 소수 3자리까지 같은 티켓이 5개 이상 쌓이면 오독이 실제 리스크가 된다

### 3.7 자산 위생
- 오업로드 방지: 평가 전용 패키지는 `work/quarantine/`으로 격리
- 디스크 100% 사고 후: 삭제/보존 목록을 명시적으로 관리 (학습 원본 dir은 zip 확인 후 삭제, fp16 원본·시드뱅크·미제출 티켓은 절대 보존)
- 데이터 덮어쓰기 사고 후: `data/`의 심링크 금지, 바이트 크기·행 수 대조

## 4. 전략 교훈 (다음 대회를 위해)

1. **초반에 σ부터 측정하라.** 시드 재추첨 2~3발을 초반에 써서 μ/σ/승자저주를 미리 알았다면, 마지막 밤의 "드로우 로터리" 5슬롯을 지터적립과 탐색에 썼을 것이다.
2. **재현성 인프라를 첫날에.** 시드 버그(torch.manual_seed가 set_seed를 덮음)를 종반에 발견 — 그 전의 모든 "시드 실험"이 무효였다.
3. **입력분포 정합 > 모델 다양화.** 이 대회의 양전 레버 상위 3개(FGM, gen_rescue, soft-F1)는 전부 "히든 입력과의 정합"이지 용량 확장이 아니었다.
4. **마감 24시간 전부터는 새 학습보다 서빙측 무GPU 레버**(지터포켓 류)가 슬롯당 EV에서 압도한다.
5. **컷이 물리적으로 닿는지 정직하게 계산하라.** 갭이 σ의 20배면 그 축은 끝난 것이다 — 희망으로 슬롯을 태우지 말고, 닿는 축(있다면)에 전부 걸어라.
6. **"불가능 부정" 지시와 정직한 확률 보고는 양립한다**: 가능성이 0이 아닌 유일 경로를 찾아 전력하되, 기대치는 숫자로 보고한다.
