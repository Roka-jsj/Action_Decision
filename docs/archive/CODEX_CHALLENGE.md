너는 Kaggle 그랜드마스터급 대회 전략가다. 아래는 진행 중인 Dacon 대회(코드제출, Macro-F1)의 현재 상태다.
내 계획의 **치명적 허점을 찾고, 더 높은 점수로 가는 계획으로 반전(invert)** 시켜라. 동의는 필요없다. 반박과 대안만 달라.

## 문제
- AI 에이전트 세션상태(구조메타+대화이력+현재발화) → 다음 action **14클래스** 예측. 지표 **Macro-F1**.
- **코드제출**: zip(model/ + script.py + requirements.txt), 서버 **T4 16GB, 오프라인, ≤1GB, 설치·추론 각 ≤10분**. 사전설치 transformers 4.46.3 / torch 2.7.1 / sklearn 1.8.0. 10제출/일. 마감 7/15.
- train 70k(세션×스텝 구조), 히든테스트 30k(세션당 1스텝, train과 세션 비공유 — 순차누출 검증→커버리지0 확인됨).

## 현재 성적/리더보드
- 내 최고 **LB 0.7400** (42위). #1 0.786, **#12 컷 0.772이고 하루 +0.016 상승중** → 최종컷 0.78~0.80 예상. 목표 0.8.
- 캘리브레이션(실측 5회): **LB ≈ holdout(스택)+bias − [0.010, 0.018]**, 오차 ±0.004.

## 검증된 사실(실측)
- 최강 단일교사: **xlm-roberta-large, 6ep, 5fold OOF 평균 0.7377** (v4직렬화 large-6ep fold0=0.7399).
- 에폭: 4→6ep +0.010/ep, 6→8ep +0.0065 (8ep fold0=0.7456, 체감).
- 직렬화 v6(이력 액션시퀀스+프롬프트패턴플래그 명시토큰 추가): **base엔 +0.018**(0.686→0.704), **large엔 무효**(−0.0008). = 큰모델은 이미 raw에서 추출.
- 멤버조합 스택(LightGBM 메타 + per-class bias 후처리):
  - klue-base(다른아키) + large: holdout+bias **0.7499**
  - base + large: 0.7501
  - base + klue + large(3멤버): **0.7531** ← 최고지만 **3멤버=1.1GB로 1GB초과 배포불가**
- 약멤버 추가 이득 미미(4멤버 +0.0014), 시드다양성 무의미(+0.001), au(5025개, 테스트는 전부 sim계열)학습제외 무효.
- 프루닝(vocab 250k→50k, id리매핑)으로 large fp16 1.12GB→0.66GB. lightgbm 벤더링으로 오프라인.

## 미사용/미검증 레버
- 증류(교사 OOF→단일 student, 현재 T=1 나이브 KD): **실측 0건**. 1GB에 3멤버+ 성능 압축하는 유일 경로로 상정중.
- Qwen2.5-0.5B(디코더계열, 완전 다른 아키): 미착수. fp16 오버플로 리스크.
- 스태커 피처: 구조스칼라+action카운트만. margin/entropy/텍스트n-gram 없음.
- FGM(구현완료 미실행), SWA/snapshot(미구현), max_len(320고정, v6@384는 −0.004로 기각).
- rank-2 정답률 57%(오답 중), hist=0슬라이스 err 44%.

## 내 현재 계획
1. M5(klue+large 2멤버, holdout+bias 0.7499) 제출 — 현 0.74와 동급, 캘리브레이션 앵커용.
2. 증류로 3멤버+ → 단일 student(1GB내) → +0.005~0.012 기대.
3. Qwen2.5 다른아키 멤버 → 스택다양성 +0.01~0.02 기대.
4. 스태커 피처업 +0.002~0.006.
→ 낙관합산 0.77~0.78, 0.8은 미봉합.

## 질문 (반박·대안 위주로)
1. **0.74→0.80 (+0.06)** 를 이 레버들로 낼 수 있나? 산술이 닫히나? 안 닫히면 뭐가 근본적으로 부족한가?
2. 내가 **과대평가/과소평가**한 레버는? 순서를 어떻게 바꿔야 하나?
3. 내가 **아예 안 본 접근**이 있나? (예: 손실함수, TTA, 프롬프트/직렬화 재설계, 세션내 멀티스텝 활용 불가지만 다른 구조신호, 대형모델 배포 다른 방법, test-time 앙상블 압축, 의사라벨/자기학습, focal/logit-adjust, 클래스 계층분류 등)
4. 컷 0.80 가정하에 남은 11일 유닛예산(~150 A100시간)을 **어디에 몰빵**해야 하나?
5. 이 문제 특성(짧은 프롬프트, 14클래스, 탐색계열 혼동, 세션당1스텝)에서 **large 단일모델의 실질 상한**은 어디라고 보나?
