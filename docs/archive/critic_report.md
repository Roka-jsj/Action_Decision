# CRITIC REPORT — 5k→LB 전이실패 부검 + 강화게이트 사전등록
표준 레드팀/품질관리. 작성 2026-07-12(powmean LB실패 직후). 앵커 은행 th85 **0.79026**, 컷(12위) 0.79283 (격차 **+0.00256**, casc필요 +0.0055~0.0071). σ_LB=0(파일결정적) 실증 — 유일 불확실성은 **전이비 β**.

## 0. 한 줄 결론 (읽고 나가야 할 것)
우리 게이트가 과대추정하는 진짜 이유는 "5k 암기공유"만이 아니다. **holdout이든 leak-free fold0이든, 둘 다 in-distribution이라 OOD 히든셋으로 전이하지 않는다.** powmean 결합기는 fold0에서 **leak-free**로 +0.00467을 냈는데도 LB −0.00096으로 반전했다 — 누수제거만으로는 부족하다는 직접증거. 전이하는 유일한 부류는 **입력분포 복원(genrescue, β=0.82)**뿐이고, 그 이유는 복원된 신호가 OOD 히든셋에도 동일하게 존재하기 때문이다. "member-quality vs decision-rule"은 잘못된 축이다. 옳은 축은 **입력수리 vs 나머지 전부**.

## (b) 강화 게이트 — 사전등록 (수정금지, 3자 서명 대기)

### 게이트 0단계 — 메커니즘 분류(먼저, 크기보다 우선)
- **(A) 입력분포 복원:** 실증된 train/serve 직렬화·토큰화·절단 불일치 수리. 검증필수: 대조군Δ≡0 + fold0 leak-free A/B 재현 + 이득이 수리행에만 암기식 집중 아님.
- **(B) member 재학습/깊이/레시피/스왑/이종백본:** 입력불변, 더 학습.
- **(C) decision-rule:** th·bias·가중·결합기·prior·router·retrieval·TTA-blend.

### 왜 ">70% P(LB>은행)"가 부류별로 갈리나 (σ_LB=0이므로 β만 불확실)
- (C): 관측 β 분포에서 **P(β>0)≈40%** → 어떤 크기도 70% 불가.
- (B) 비수리: in-dist 이득이 암기면 부호반전, 제출 **0/4 양성** → P(LB>은행)≤25%.
- (A): 제출 1/1 양성, β 0.5~0.82 → 유일하게 70% 통과 가능 부류.

### 사전등록 임계값
**(A) 입력수리 — 은행 GO 조건 (전부 충족):**
1. leak-free fold0 캐스케이드 Δ **≥ +0.0015**
2. 멤버 solo ΔF1 (leak-free) **≥ +0.003**
3. 대조군(비수리행) Δ = 0 (byte)
4. 이득 GEN삭제/암기행 집중 **아님**
5. 30k 리플레이 시간 **≤ 555s**
→ 예상 LB = casc × [0.5,0.82]. P(LB>은행)>0.8.

**(B) member 비수리 — 은행 부적격, 프로브 전용:**
- 반드시 **leak-free fold0-OOF**(멤버가 eval fold 미학습). **5k-holdout 금지**(FULL 암기).
- solo ΔF1(leak-free) **≥ +0.008** AND 캐스케이드 Δ(leak-free) **≥ +0.006**.
- **변경행 검사**(m1t3/mdeb-T3 사망서명 차단): 후보가 argmax를 **바꾸는** 행에서 leak-free F1 **≥ 0**. (게이트C/E 부검: 변경행 −0.015 균일결손이 사망원인.)
- 이득 GEN/암기행 집중 아님.
- **fold0만으로 은행돌파 선언 금지** — σ_LB=0 무하방 프로브로만.
- 산술상한: leak-free casc × [0.12, 0.42]. **컷(+0.00256) 도달하려면 casc ≥ +0.006@β0.42 또는 ≥ +0.021@β0.12** → 사실상 (B)는 컷 미도달. (B) = 프로브지 컷경로 아님.

**(C) decision-rule — 은행 GO 없음.** 실측 0/12 클리어. 비선형 전부 음. 제출 금지(결합기 실패가 정준증명).

## (c) mdeb14 GO/NO-GO — 들어올 fold0 수치 사전등록

**대상:** teacher_mdeb14_f0.npz(leak-free, 학습중) vs mdeb12ep_f0.npz(배포mdeb, w=**0.40** 대형멤버). m2-swap 캐스케이드 델타 = **C_lf**.

**분류: (B) 비수리(14ep vs 12ep 깊이차, 입력불변).** 따라서 (B) 게이트 적용.

**인플레 진단:** 5k solo +0.01394 / casc +0.00598은 **비대칭 암기** — mdeb14(14ep)가 공유 5k를 배포mdeb(12ep)보다 2에폭 더 암기. m1swap β(0.36)는 **동깊이**에서만 유효하므로 5k +0.00598에 직접 적용 금지(그래서 leak-free teacher 필수). mdeb는 ep12서도 상승중(mdeb-T3 프로브 0.7640 미수렴)이라 14ep가 fold0-val을 **일부 진짜 개선**하나, +0.006의 대부분은 2에폭 추가암기. **예측: C_lf ≈ +0.002~+0.004 (인플레 50~65%).**

**사전등록 분기:**
| C_lf (leak-free fold0 casc) | 판정 | 근거 |
|---|---|---|
| **< +0.004** | **NO-GO** | 미라지 확정. +0.006은 암기인플레. 조립중단. |
| +0.004 ~ +0.007 | **MARGINAL → NO-GO(은행)** | 프로브만 허용, 단 변경행 F1≥0 ∧ 비GEN집중 ∧ 여유슬롯 ∧ 상위EV 없을때. 예상LB +0.0005~+0.0018(β0.12~0.36) — **컷 미도달**. "오직 컷" 목적함수하 기본 NO-GO. |
| **≥ +0.007** ∧ 변경행F1≥0 ∧ 비GEN | **GO(고분산 프로브)** | 예상LB ≤ +0.0025@β0.36 — β 상단 착지시 컷 근접 가능. mdeb14가 정당 컷베팅 되는 유일 시나리오. |

**질문 직답 — "+0.006 인플레가 leak-free +0.002~0.003이면 LB 양전할까?":** +0.002~0.003 leak-free는 β 0.12~0.36에서 LB **+0.0003~+0.0011** — 양수일 **가능성은 있으나** 컷 미달이고 (B)부류 반전리스크 밴드 내. **은행제출 부적격, 무하방 프로브 가치도 "오직 컷" 목적함수하 없음.** 가장 가능성 높은 결과: mdeb14는 NO-GO 또는 marginal 프로브. 컷경로 아님.

- **decision-rule 축은 완전소진·전이 음수 확정**(결합기가 마지막 증명). 재론 금지.

### 최고확률 LB상승 경로 (정직)
**컷(+0.00256)에 대한 >30% 경로는 없다.** M4(최대 헤드룸)는 7아키텍처 수렴·프롬프트단독 F1 0.23·동일프롬프트 80.5% 상충라벨로 **알레아토릭 확정**. 컷 필요조건(M4개선)이 물리 불가.

**유일하게 β>0.5로 전이하는 메커니즘 = 입력분포 복원(genrescue 부류).** 따라서 단 하나의 최고확률(그래도 낮은) 경로:
> **잔여 train/serve 불일치 사냥.** 배포 추론의 토큰화·특수토큰·**좌측절단** 로직이 히든셋 입력형태에서 학습시와 EXACT 일치하는지 바이트검증. [GEN]·left-truncation처럼 아직 안 잡힌 직렬화 갭이 있으면 그게 genrescue급 레버(β 0.5~0.82). 대조군Δ≡0 + fold0 A/B로 게이트(A) 통과시에만 제출.

그런 수리가 **없다면** 정직한 입장: **은행 th85 0.79026 유지·순위방어(16~21위 0.0005폭 밀집), top-12는 측정경로 밖.** 결합기·member·decision-rule에 추가 슬롯·GPU 투입은 음의 기대값.

### 캠페인에 걸 규율 3조
1. **메커니즘 먼저 분류**(A/B/C) 후에만 크기 논의. 5k/fold0 숫자를 β 없이 인용 금지.
2. **모든 member 판정은 leak-free fold0-OOF**. 5k-holdout(FULL 암기)로 member 판정 영구금지.
3. **비선형 decision-rule은 제출 전 자동기각**(결합기·2단·prior·dual·router 실측 전부 음).
