# 마감일 슬롯 경제학 + 최종선택 불변식

## 슬롯 배분 원칙
1. **티켓별로 P(목표컷 도달)과 P(전역최대 경신)·기대폭을 분리 계산** — 이 둘은 다른 지표다. 컷이 도달권이면 컷꼬리(σ 큰 티켓)에, 도달 불가면 경신확률(σ 작아도 μ 높은 티켓)에 배분.
2. **버림서열 사전 확정**: 쿼터가 줄어들 때 무엇부터 버리는지 순서를 미리 등록 (여유분 → 저EV 필러 → 중복성 티켓 → ... ). 판독 직후 즉흥 결정 금지.
3. **"P(초과)만 비교" 함정**: 경신확률 50%짜리 티켓이라도 기대폭이 ~0이면 12%×기대폭 있는 티켓에 열위일 수 있다 — 확률×폭×효용으로.
4. **컷 도달 가능성 정직 계산**: 갭/σ ≥ 4면 그 축은 운영상 종료. 희망으로 슬롯을 태우지 말고 도달 가능한 유일 경로(있다면)에 몰빵하되 기대치는 숫자로 보고.
5. **예비 슬롯 규율**: 업로드/런타임 오류 대비 예비를 마감 T−10분까지 보존 — 단 "지킬 것"(이미 확보된 점수)이 없으면 예비는 무가치, 전량 발사가 옳다.

## 시간 역산 (하드컷 사전등록)
- `마지막 업로드 컷 = 마감 − 채점시간 − 여유` (채점시간은 과거 제출의 소요시간 실측으로)
- 조립 파이프라인 소요(양자화+zip+캐너리)를 실측해 `조립 착수 데드라인` 도출
- 마감 직전 발사분은 "점수 도착이 선택 마감 이후"가 되지 않는지 확인 — 채점 못 받는 제출은 무가치

## 최종선택(체크박스) 불변식 — 가장 값싼 보험
1. **지금 즉시 선체크**: 마감 직전 1회 선택은 단일창구 리스크. 현 전역최대에 미리 체크하고, 이후 "체크박스는 항상 현재 전역최대를 가리킨다"를 불변식으로 유지. 마감 직전은 선택이 아니라 **검증** 시점.
2. **3중 대조**: 스왑은 제출ID + 파일명 + 풀 자릿수 점수 문자열 일치 확인 후에만. 점수들이 소수 3자리까지 같아지는 종반엔 오독이 실제 사고 벡터. "최신 행"이나 "정렬 최상단"으로 고르기 금지.
3. **엄격초과만 스왑**: Public=Private(셰이크업 없음)이면 +1e-10이라도 초과면 스왑, 완전동점이면 현직 유지(조작 리스크만 추가).
4. **지속성 검증**: 체크 후 새로고침 → 상태 재확인 → 스크린샷. 플랫폼이 저장 실패를 조용히 삼킬 수 있다.
5. **no-swap-after-T**: 마감 2~3분 전부터는 예외조건(엄격초과 + 2분 내 검증 가능) 아니면 무접촉.
6. 최종선택 2개 허용 플랫폼이면 2번 슬롯에 차순위 점수를 DQ 보험으로.
