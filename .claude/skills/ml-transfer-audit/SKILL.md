# 오프라인→실전 전이 감사

오프라인 지표 상승이 히든/실전 점수로 이어지는지 기전 단위로 감사한다. 핵심 원칙: **"오프라인 +는 실전 +가 아니다. 기전 클래스가 전이를 결정한다."**

## 절차

1. **기전 분류** — 제안된 개선을 아래 클래스로 분류:
   - **Class-R (강건화·지표정합)**: 적대학습(FGM/AWP), 평가지표 서로게이트 손실(soft-F1 등), 정칙화 → **전이 경향 높음**
   - **Class-A (입력분포 정합)**: 절단 복원, 직렬화 수리, 전처리를 히든 입력분포에 맞춤 → 전이하나 **1회성·소진형** (같은 축의 2번째 개선은 급감)
   - **Class-M (적합 강화)**: 에폭 증가, 증류, warm-start, 더 큰 모델로 같은 데이터 재적합 → **역전 경향** (in-dist 과적합이 OOD를 훼손)
   - **Class-C (결정규칙)**: 앙상블 가중치·게이트 임계·결합기·bias 재적합 등 홀드아웃에 맞춘 후처리 튜닝 → **전이 최악** (본 캠페인 0/12 전멸)

2. **원장 대조** — 프로젝트의 실험 원장(experiments_master.csv 등)을 grep해 같은 계열의 과거 실측 판정이 있는지 확인. 사망 판정 계열의 재개봉은 오프라인 수치가 아무리 좋아도 금지.

3. **전이계수 추정** — 같은 클래스의 과거 실측 (오프라인Δ → 실전Δ) 쌍에서 β를 추정해 기대 실전Δ = β × 오프라인Δ 계산. β 미상이면 Class-R 0.3~0.5, Class-A 0.5~0.8(첫회), Class-M/C 0 이하로 보수 가정.

4. **판정 출력** — SUBMIT(기대Δ>0이고 슬롯가치 있음) / HOLD(정보가치만) / DEAD(사망계열 또는 기대Δ≤0). 근거 1줄과 함께 원장에 기록.

## 함정 목록
- 홀드아웃 +0.005급도 Class-C면 실전 음전 가능 (실측 사례 다수)
- in-dist 평가는 버그 판별용이지 후보 선별용이 아님 (역상관 실측 사례 있음)
- 단일 측정으로 판정 금지 — 노이즈(σ) 대비 효과크기 먼저 (관련: /seed-sigma-first)
