# GPU 학습 함대 감시·운영 수칙

## 감시 설계 (Monitor/백그라운드 폴링 루프)

감시 대상 4종을 모두 커버해야 한다 — 침묵은 성공이 아니다:
1. **완주**: 산출물(member zip 등) 출현 — 단 쓰기중 레이스 주의(크기 안정 + testzip 후 사용)
2. **사망**: `ps -o stat= -p $PID`가 GONE 또는 Z — 완주 마커 로그 유무로 DONE/DEAD 구분
3. **정지(stall)**: 로그 mtime이 에폭시간의 1.5배 이상 정지
4. **환경 붕괴**: `torch.cuda.is_available()` 주기 폴링 (NVML 붕괴 시 신규 프로세스만 CUDA 불가, **기존 학습은 생존 가능** — 붕괴해도 재시작 전에 완주를 기다리는 게 정답일 수 있다)

함정: `pgrep -f "MY_ENV_VAR=x"`는 환경변수를 못 본다(커맨드라인만 매칭) → **PID 기반으로 감시**. PID는 발사 시점에 기록.

## 함대 운영 수칙 (전부 실사고 기반)

- **kill은 반드시 PID 지정**. 패턴 kill 금지.
- **이중발사 감지**: 같은 태그 프로세스 2개 = GPU를 반씩 나눠 2배 느려짐. `/proc/PID/environ` 덤프로 정체 확인(`tr '\0' '\n' </proc/$PID/environ | grep TAG`) 후 후발 제거 — 동일 시드면 가치 0인 완전 중복.
- **발사 후 90초 생존확인**을 발사 스크립트에 내장 (OOM·즉사 조기 검출).
- **자동 큐는 마감 임박 시 위험**: busy() 판정 결함으로 학습 중인 GPU에 오발사한 실사고. 마감 24시간 전부터는 자동 큐를 죽이고 수동 원샷 스크립트로 전환 (CUDA 프로브 → 기존 학습 잔존 체크 → 발사 → 생존확인).
- **에폭 자동룰**: 마감 역산으로 가능한 최대 에폭 계산, 못 맞추면 발사 취소를 사전등록. `가용분 = (조립컷시각 − 지금)/60 − 여유25분; E×페이스 ≤ 가용분`.
- **컨테이너 재시작 후**: CLI 심링크(확장 업데이트로 target 이동), 셸 PATH, 자동화 프로세스 전부 재검증.
- **학습 로그에 에폭당 loss 1줄은 반드시 계기로**: 없으면 사후 법의학이 불가능하다 (실사고: 원인불명 -0.005를 로그로 판별 못함).

## 판별실험 원칙
이상 징후(점수 급락 등)에 완벽히 상관된 요인을 찾아도 **인과 아님** — 실사고: gradient checkpointing이 저점수 런들과 100% 상관했으나 클린 대조실험 2발로 무죄 판명. 대조실험(단일변수 분리) 전에 범인 확정 금지.
