# SR 재양자화 지터 티켓 (무GPU 재추첨)

## 원리
int8 양자화(per-row/group RTN)는 결정론적 반올림이라 하나의 "실현"에 불과하다. fp16 원본에 **확률적 반올림(SR, 비편향)** `q = floor(x + u), u~U(0,1)`을 적용하면 int8 엔트리의 ~25%가 ±1LSB 플립 → 저마진 예측 ~3%가 플립 = **원본과 기대성능이 같은 새로운 추첨 티켓**. RTN 실현이 불운했다면 공짜로 회수한다.

실측 (한 캠페인, 30k 히든 macro-F1): 8발 전부 원본 RTN 대비 −0.0006~+0.0005 밴드, σ≈0.0002, 최종 +0.0005 회수. **m1+m3(두 멤버) 복합 지터 > 단일 멤버 지터** (상위 3점 전부 복합).

## 구현 요점
```python
# quantize 스크립트에서 np.round(x) 대신:
rng = np.random.default_rng(qseed)          # qseed가 티켓 ID
q = np.clip(np.floor(x + rng.random(x.shape, dtype=np.float32)), -127, 127).astype(np.int8)
# 증폭판(폭 k>1): z=(rng.random(...)-0.5)*k; q=np.round(x+z)  ← 단 μ비용 발생, k=1이 최적(실측)
```

## 운용 규칙 (실측 기반)
- **k=1(순수 SR)이 최적**: k=2는 μ −0.0002급 비용, k≥3 금지
- **앙상블이면 모든 양자화 멤버를 복합 지터** (단일 멤버 지터의 σ 기여는 멤버 가중치에 비례해 작음)
- 좋은 티켓이 나오면 그 티켓의 **행운 멤버는 고정하고 나머지 멤버만 재지터** — 경신확률 최대 구성 (실측: 마지막 발 ~50% 확률 적중)
- σ≈0.0002 수준이므로 **큰 갭(>4σ)을 메우는 수단이 아니라 미세적립·승자저주 회수 수단** — 컷 도달용으로 슬롯 배정 금지
- 티켓마다 byte-diff 가드(원본과 동일하면 슬롯 낭비) + 런타임 캐너리 필수
- fp16 원본 zip을 반드시 보존해야 이 축이 열린다 — 배포 파이프라인 설계 시 원본 보존을 기본값으로

## 언제 쓰나
- GPU는 꽉 찼는데 제출 슬롯이 남는 시간대 (밤샘 학습 대기, 마감 직전)
- 새 학습이 물리적으로 불가능한 마감 수 시간 전
- 배포본이 오래된 단일 양자화 실현일 때 (재추첨 1~2발은 거의 항상 +EV)
