# 판독 프로토콜 (점수가 나올 때마다)

## 평시 절차
1. **원장 append**: 점수·제출ID·구성·맥락을 실험 원장(experiments_master.csv 류)에 즉시 기록. 원장은 단일 진실원천 — 이후 에이전트들이 사망계열 재제안을 자동 차단하는 근거.
2. **사전등록 분기 집행**: 판독 전에 등록해둔 분기표("X 이상이면 A, 이하면 B")를 기계적으로 실행. 분기표가 없었다면 다음 판독 전에 만든다 — 판독 후 즉흥 결정은 감정 이탈의 온상.
3. **다관점 판정**: 규모에 따라 서브에이전트 패널(2~4관점: 해석/재계획/반증) 또는 최소한 "이 판독이 기존 가설 중 무엇을 죽였나"를 명시. 판정은 SUBMIT/HOLD/DEAD + 기대Δ + 근거 스키마로.
4. **계획 델타만 보고**: 기존 계획 대비 변경분(없으면 "유지")을 명시.

## 충격판독 위기 프로토콜 (예상 밖 점수)
순서 고정 — 공황 행동 방지:
1. **즉석 무결성 검사** (분 단위): 제출물 해시/메타/구성요소가 의도와 일치하는지. "무엇이 제출됐나"부터.
2. **저비용 판별 프로브**: 라벨 있는 소형 세트(예: 500행)로 제출물을 로컬 실행 — 기계적 버그(모델 무력화·경로 오류)와 품질 문제를 분리. 버그면 수리, 품질이면 다음 단계.
3. **분리 실험**: 의심 요인을 단일변수로 바꾼 대조 조립·평가 (예: 스캐폴드만 교체).
4. **가설 명시 + 사전확률 + 업데이트 규칙**: T1 버그 / T2 실력 / T3 환경 / T4 상호작용 식으로 나열하고, 각 프로브 결과가 어느 가설을 얼마나 움직이는지 **결과 도착 전에** 등록.
5. **판별 전 추가 제출 동결**, 진행 중 학습은 중단하지 않음(증거 수집 계속).

## 이례 신호 감지 체크리스트
- 서로 다른 두 모델의 점수 동률(Δ<노이즈/10) → 우연확률 계산 후 수사
- 연속 단조 상승/하락 → 순서통계 확률(n연속 = 1/n!)부터
- 오프라인 순위와 실전 순위 역전 → in-dist 선별 중단 신호
